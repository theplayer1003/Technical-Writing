# 우아한테크코스 프리코스 과제 제출 가이드

## 목표
- 우아한테크코스(이하 우테코) 프리코스 과제 수행을 위한 Git 사용법을 공부한다
- 우테코 `GitHub` 의 미션 저장소를 개인 계정으로 복사하고, 기능을 구현한 뒤 `Pull Request` 하는 과정을 공부한다

## 사전 요구사항
- `GitHub` 계정 생성 및 로그인
- `IntelliJ` 등의 개발 환경 설치
- `Git` 설치 및 터미널 사용 환경 설정

## 단계별 가이드

### 전체 작업 흐름
1. **Fork** : 저장소 복제
2. **Clone** : 저장소를 로컬 환경에 복제
3. **Branch** : 작업 공간 분리
4. **Commit** : 작업 단위별 구현 내용 저장
5. **Push** : 결과물을 원격 저장소에 업로드
6. **Pull Request** : 코드 리뷰 요청(과제 제출)

### fork
- 제공된 과제는 '우테코의 저장소' 에 있으며 이를 '본인 계정의 저장소' 에 복사본을 생성해서 작업해야 합니다.
1. 우아한테크코스 원격 저장소 페이지에 접속한다.
2. 수행해야 할 미션 레포지토리 우측 상단의 **Fork** 버튼을 클릭해 본인 계정으로 복제한다.

### clone
- `GitHub` 에 있는 저장소를 내 컴퓨터, 로컬 환경으로 다운로드해 개발 환경을 구성합니다.
1. **Fork** 완료 한 본인 계정의 저장소에서 URL을 복사한다.
2. 터미널에서 작업하고자 하는 폴더로 이동한다.
3. 아래 명령어를 입력한다.
```bash
# 저장소의 주소 양식은 다음과 같다
git clone https://github.com/{본인_계정명}/{저장소이름}.git
```

### branch
- 다운로드 한 원본에 영향을 주지 않고 독립적으로 기능을 구현하기 위해 새로운 브랜치를 생성합니다.
- 사용하는 IDE 를 실행하고 터미널을 켜서 명령어를 실행합니다.
```bash
# 브랜치 생성 후 해당 브랜치로 즉시 전환하는 명령어
git switch -c {생성할_브랜치명}
```

>[!note]
>왜 `git switch`를 사용하나요?
>
>기존에 많이 사용되던 `checkout` 명령어는 '브랜치 변경'과 '파일 복원' 두 가지 기능을 모두 수행하여 혼란을 줄 수 있습니다. Git 2.23 버전부터는 이를 명확히 구분하기 위해 브랜치 조작 전용인 `switch`가 도입되었습니다. (우테코 과제에서는 어느 것을 사용해도 무방합니다.)

### add
- 작성한 코드 중 저장할 파일을 선택합니다.
```bash
# 모든 파일을 선택
git add .

# 특정 파일만 선택 가능
git add ./src/.../{파일명}.java
```

### commit
- `add` 로 선택된 파일들을 하나의 작업 단위로 확정하고 설명 메시지를 남깁니다.
- 우테코에서는 \[Angular JS Git Commit Message Conventions] 을 권장 합니다.
```bash
git commit -m "{커밋을 설명할 수 있는 메시지}"
```

### push
- 로컬 환경에서 작업한 결과물을 본인의 `GitHub` 저장소에 업로드 합니다.
```bash
git push origin {작업_브렌치명}
```

### pull request
- 본인 계정의 저장소에 업로드된 결과물을 우테코 원본 저장소에 반영해 달라고 요청합니다.
- 프리코스는 이 과정을 **과제 제출 행위** 로 활용하고 있습니다.
1. 본인 계정의 GitHub 저장소 페이지에 접속한다.
2. 상단의 Compare & pull request 버튼을 클릭한다.
3. 양식에 맞춰 PR 게시글을 작성하고 Create pull request 를 클릭해 제출한다.
